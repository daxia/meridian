## 贡献指南

### 命名规范
- **Bug 修复**: `fixbug-[模块]-[简述]-[日期].md`
- **经验总结**: `EXP-[模块]-[简述]-[日期].md`

### 模块标识
- `WEB`: 前端 (Nuxt/Vue)
- `API`: 后端 (Hono/Workers)
- `SYS`: 系统/基础设施/DevOps
- `ML`: 机器学习/算法
- `DB`: 数据库/SQL

### 模板建议
建议包含：
1.  **问题描述**: 现象、报错信息。
2.  **根因分析**: 为什么会发生。
3.  **解决方案**: 如何修复（代码片段）。
4.  **经验教训**: 如何避免再次发生。
