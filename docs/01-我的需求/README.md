# 01-我的需求 目录说明

- **唯一需求文档**：**我的需求.md** 为需求方（用户）填写的原始需求唯一集合。**新需求一律在此文档中按条追加**，并标注状态（待实现/进行中/已完成/废弃/未来实现）。
- **模板**：`00-REQ-我的需求模板.md` 提供单文档结构与状态约定，供参考或复制。
- **历史文档**：旧版需求已归档至 **history/** 子文件夹，不再在根目录新增单需求文件；若需为某条历史需求写 PRD，可在 我的需求.md 中追加一条并注明来源后，再走 PRD → 设计 → 实现流程。
- **审核与疑问**：需求审核、疑问提出与回答、审核结论均在 **02-产品需求文档PRD** 的对应 PRD 中完成，不在此文档填写。
