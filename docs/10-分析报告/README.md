## 报告类型

1.  **可行性分析 (Feasibility)**: `REQ-[模块]-[主题]分析报告-[日期].md`
    - 评估新技术引入或复杂功能实现的成本与风险。
2.  **故障根因分析 (RCA)**: `Report-[主题]-[日期].md`
    - 对生产事故或严重 Bug 进行复盘，寻找根本原因。
3.  **性能分析 (Performance)**:
    - 针对系统瓶颈的测试数据与优化方案。

## 核心内容结构

1.  **概述**: 分析的目标与背景。
2.  **现象/数据**: 收集到的事实、日志截图、监控指标。
3.  **分析过程**: 逻辑推理、排查步骤、代码审计。
4.  **结论与建议**: 明确的下一步行动计划（Action Items）。
